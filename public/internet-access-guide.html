HarvestLoop Internet Access Guide

Options for making your application accessible from different networks

Understanding the Challenge

When running your HarvestLoop application on a local machine, it's only accessible to devices on the same network by default. This is because:

  • Your server is running on a private IP address (like 192.168.x.x or 10.x.x.x)
  • These addresses are not routable across the internet
  • Devices on different networks cannot directly reach each other

Solution Options

Option 1: Port Forwarding (Home Setup)

If you have control over your router, you can set up port forwarding:

  1. Access your router's admin panel (usually at 192.168.1.1 or 192.168.0.1)
  2. Find the "Port Forwarding" section
  3. Create a new rule that forwards external port 3000 to your computer's internal IP and port 3000
  4. Use a service like whatismyip.com to find your public IP
  5. Others can now access your application at http://YOUR_PUBLIC_IP:3000
Note: Your ISP may change your public IP periodically. Consider using a dynamic DNS service.

Option 2: Cloud Hosting (Most Reliable)

For production use, hosting your application on a cloud platform is recommended:

  1. Sign up for a cloud platform (AWS, Azure, Google Cloud, DigitalOcean, Heroku, etc.)
  2. Deploy your application to the cloud platform
  3. Configure your database connection in the cloud
  4. Set up a domain name to point to your cloud service

Popular affordable options for Node.js applications:

  • Railway - Easy deployment, free tier available
  • Render - Simple setup, free tier available
  • Netlify - Great for static frontend + API

Option 3: Tunneling Services (Quick Testing)

For temporary access or testing, tunneling services can create a public URL:

  • Ngrok - Popular option but requires account setup
    npm install -g ngrok
    ngrok http 3000
  • Cloudflare Tunnel - Reliable and secure
    npm install -g cloudflared
    cloudflared tunnel --url http://localhost:3000
  • Localtunnel - Simple to use
    npm install -g localtunnel
    lt --port 3000

These services create a temporary public URL that forwards to your local server.

Note: Free tiers of these services often have limitations on bandwidth and connection time.

Recommended Approach

For your HarvestLoop application, we recommend:

  1. Development: Continue using local network for testing between devices on the same network
  2. Testing across networks: Use a tunneling service like Ngrok for temporary access
  3. Production: Deploy to a cloud platform for reliable, permanent access

For most small to medium businesses, affordable cloud hosting options start around $5-10/month and provide much more reliability than trying to host from a personal computer.

Cloud Deployment Quick Guide

Here's a simplified guide to deploy HarvestLoop to Railway (one of the easiest platforms):

  1. Create a GitHub repository for your HarvestLoop project
  2. Push your code to GitHub
  3. Sign up at Railway.app (free tier available)
  4. Click "New Project" â†’ "Deploy from GitHub repo"
  5. Select your HarvestLoop repository
  6. Add a MongoDB database from the Railway dashboard
  7. Set environment variables (MONGODB_URI, etc.)
  8. Railway will automatically deploy your application
  9. You'll get a public URL that works from any network
Pro Tip: You can add a custom domain name to make your application more professional.
